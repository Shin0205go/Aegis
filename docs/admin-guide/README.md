## 🎯 管理者の責務

### 1. システム保守
- AEGISの安定稼働維持
- 定期的なアップデート適用
- パフォーマンス監視と最適化

### 2. セキュリティ管理
- アクセス権限の適切な管理
- セキュリティパッチの適用
- 監査ログの定期レビュー

### 3. ポリシー管理
- ポリシーの承認プロセス運用
- 定期的なポリシーレビュー
- ユーザーフィードバックの反映

### 4. ユーザーサポート
- ユーザーからの問い合わせ対応
- トレーニングの実施
- ドキュメントの更新

## 💡 クイックリファレンス

### 重要なコマンド
```bash
# システム状態確認
npm run status

# ログローテーション
npm run rotate-logs

# バックアップ実行
npm run backup

# 診断レポート生成
npm run diagnostics
```

### 重要なファイル
- `.env` - 環境変数設定
- `aegis-mcp-config.json` - MCPサーバー設定
- `policies/` - ポリシーファイル
- `logs/` - 各種ログファイル

### 監視URL
- ヘルスチェック: http://localhost:3000/health
- メトリクス: http://localhost:3000/metrics
- 管理API: http://localhost:3000/api/admin

## 🚨 緊急時対応

### システム停止時
1. プロセス状態を確認
2. ログを確認してエラーを特定
3. 必要に応じて再起動
4. [トラブルシューティング](./troubleshooting.md)を参照

### セキュリティインシデント
1. 影響範囲を特定
2. 必要に応じてアクセスを一時停止
3. 監査ログを保全
4. インシデント報告書を作成

## 📈 運用のベストプラクティス

1. **定期メンテナンス**
   - 週次: ログレビュー、メトリクス確認
   - 月次: ポリシーレビュー、アップデート確認
   - 四半期: セキュリティ監査、パフォーマンス評価

2. **変更管理**
   - すべての変更を記録
   - 本番適用前にテスト環境で検証
   - ロールバック計画を準備

3. **ドキュメント管理**
   - 設定変更をドキュメント化
   - 運用手順書を最新に保つ
   - ナレッジベースの構築
