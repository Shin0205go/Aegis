# AEGIS ドキュメント構造ガイド

このガイドは、AEGIS プロジェクトのドキュメント体系を維持しやすくするための運用メモです。ドキュメントの大半はすでに役割別に整理されており、新しい情報を追加するときも同じ整理基準を守ることで、利用者が迷わず目的の資料に辿り着けます。

## 🎯 ガイドの目的

- ルート `README.md` を簡潔に保ち、プロジェクトの概要と導線に集中させる
- `docs/` 配下の役割別ガイドを明確に分離し、対象読者ごとに最適な情報を提供する
- 新しいトピックを追加する際のチェックリストを用意し、リンク切れや情報の重複を防ぐ

## 📁 ドキュメント全体像

```
docs/
├── README.md                  # ドキュメントのホーム（総合目次）
├── introduction.md            # AEGIS の背景や価値の紹介
├── odrl-testing-guide.md      # ODRL ハイブリッド判定のテスト手順
├── ODRL_INDEX.md              # ODRL ドキュメントの目次
├── ...（その他 ODRL 関連ドキュメント）
├── user-guide/                # エンドユーザー向けガイド
├── admin-guide/               # 管理者向けガイド
├── developer-guide/           # 開発者向けガイド
└── reference/                 # 用語集・ロードマップなどのリファレンス
```

### ルート `README.md`

- プロジェクトの概要紹介と主要リンクに限定する
- 詳細な手順や長文の説明は `docs/` 以下に移し、リンクで誘導する
- 主要な導線（ユーザー・管理者・開発者・ODRL）へのリンクを最新の状態に保つ

### `docs/README.md`

- 役割別ガイドへの目次として機能させる
- 新しいトピックを追加した際は、対象読者のセクションに必ず追記する
- 長くなりすぎないよう、各リンクの説明は 1 行程度にまとめる

## ✅ 追加・更新時のチェックリスト

1. **配置場所の確認** – 対象読者（ユーザー/管理者/開発者/リファレンス/ODRLなど）に応じて配置場所を選択する
2. **相互リンクの更新** – `README.md` や `docs/README.md` に必要なリンクを追加し、既存リンクの整合性を確認する
3. **関連資料の参照** – 既存の類似トピックがないか `rg` などで検索し、重複や矛盾を避ける
4. **リンク切れの防止** – 相対パスを使用し、ファイル名の変更があれば他のドキュメントも必ず更新する
5. **レビュー時の観点共有** – PR 説明に「どのガイドを更新したか」「読者にとっての改善点」を明記する

## 🔄 継続的な改善のヒント

- 定期的に `docs/` 配下を読み返し、古い前提や用語が残っていないか確認する
- 大きな構成変更を行う際は、このガイドに方針を追記し、次回以降のメンテナンスに活かす
- ドキュメントの更新履歴を残したい場合は `docs/reference/changelog.md` に要約を追加する
